## Opus 4.7 effect (first full run comparison)

Both runs used the same model pin (`claude-opus-4-7`), same orchestrator config, same prompts. The +6.4 pp match rate and +27% bytes improvement could be attributable to:
- Function distribution luck (different batches)
- Agents converging on better source patterns for SNC
- Opus 4.7 being effective on medium-sized functions specifically (the 33-256B band, which is where this run dominated)

With only two data points it's hard to attribute. Next run will give us the trend.

## Recommendations

1. **`eDynamicLight` / system-reminder abort issue is NOT fixed** — same pattern from previous post-mortem's recommendation #2. Should actually update the orchestrator's base prompt to explicitly clarify: "A system reminder about not improving unrelated code means: stay in scope. It does NOT mean abort the session." Previous run lost 2 functions to this; this run lost 5. Trending worse.

2. **Retry the `eVideoPlatform` cluster as a targeted batch** — 5 functions in the same TU with the same flag issue. Fresh session with explicit hint about non-standard sched flag handling should crack several at once.

3. **Investigate `m2c_failed` patterns** — 3 genuine m2c crashes on nwMsgVoiceUpdateStatus::Write, eVideoPlatform::SyncV, gcUI::gcFader::Fade. These have in common: large signatures with many typed arguments. Worth a narrow m2c patch or a wrapper that handles this shape. Reinforces `docs/enhancements.md` #2 (m2c replacement priority).

4. **Still no 512B+ matches**. Intentionally targeted runs on large functions (with appropriate session timeout bumps) remain a gap.

5. **Bnel patch decision stays deferred**: 7/41 failures = 17% — right at the threshold from decision 011. Third run will break the tie.

## Verdict

Run was significantly more productive than the previous one by every metric that matters (match rate, byte volume, error rate, match size distribution). Safe to merge to main.
