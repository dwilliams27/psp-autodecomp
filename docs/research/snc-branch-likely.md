# SNC Branch-Likely Generation Rules

Reverse-engineered from SNC compilation experiments (2026-04-14, 89 tool uses).

## What Are Branch-Likely Instructions?

MIPS branch-likely variants (`beqzl`, `bnezl`, `beql`, `bnel`, `blezl`, `bgtzl`) execute the delay slot instruction ONLY if the branch is taken. If not taken, the delay slot is nullified.

SNC uses these extensively — 2,802 functions in the EBOOT have branch-likely instructions.

## 7 Patterns That Generate Branch-Likely

### Pattern 1: Null-check-and-dereference
```cpp
if (ptr != NULL) { x = ptr->field; }
```
→ `bnezl $ptr, target` with `lw $x, offset($ptr)` in delay slot. The load only executes if ptr is non-null.

### Pattern 2: Array count after null check
```cpp
int count = 0;
if (arr != NULL) { count = arr[-1]; }  // cArray pattern
```
→ `bnezl $arr, +1` / `lw $count, -4($arr)`

### Pattern 3: Short-circuit evaluation
```cpp
if (a && b) { ... }
```
→ `beqzl $a, skip` (skip evaluation of b if a is false)

### Pattern 4: Loop with preloaded next iteration
```cpp
while (node != NULL) {
    process(node);
    node = node->next;
}
```
→ `bnezl $node, loop_top` / `lw $field, offset($node)` — preloads a field for the next iteration in the delay slot.

### Pattern 5: Conditional assignment with side effect
```cpp
if (result != 0) { flag = 1; }
```
→ `bnezl $result, +1` / `li $flag, 1`

### Pattern 6: Multi-field comparison (lexicographic)
The `cRedBlackTree::FindNode` pattern — comparing 5 fields lexicographically:
```cpp
if (a.f1 < b.f1) return -1;
if (b.f1 < a.f1) return 1;
// equal, load next field
if (a.f2 < b.f2) ...
```
→ `beqzl $cmp, +3` / `lw $next_field, offset($key)` — loads the next comparison field only if the current comparison was equal.

### Pattern 7: Type-check with cast
```cpp
if (type == EXPECTED) { derived = (Derived*)ptr; use(derived); }
```

## The bnel Divergence (NOT These Patterns)

The documented `bnel` divergence (docs/decisions/006-bnel-compiler-divergence.md) is a SEPARATE issue. It affects specifically:
- Two-register `bnel` with `move`/`li` in the delay slot
- Cases where our SNC version disagrees with the original on whether to use branch-likely

The 7 patterns above are RELIABLY generated by both compiler versions.

## sched=1 vs sched=2 Effect

Minimal effect on branch-likely generation. Both sched modes produce the same branch-likely decisions. The difference is in how surrounding instructions are scheduled, not in the branch form choice.

## Matching Implications

1. Write C++ that naturally produces the null-check-and-dereference pattern — don't try to force it with asm
2. For lexicographic comparison (FindNode), write inline comparison functions that compare field-by-field
3. Loop patterns with `bnezl` can be matched with `while(node)` loops that access a member early
4. The key is producing code where the delay slot instruction is ONLY useful when the branch is taken
