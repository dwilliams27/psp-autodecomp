# 001: Target Selection Strategy

## Context

The system needs a proof-of-concept target with known ground truth, then a graduation path to commercially significant games. The ideal progression maximizes pipeline reuse between milestones — same engine, same compiler, same tooling — so each step validates more of the system without requiring a tooling rewrite.

## Decision

### Milestone 1: Days of Thunder (USA, PSP Mini)

Days of Thunder is a ViciousEngine PSP Mini published by Sony Online Entertainment (2009). It ships with `.sym` and `.map` files inside `PSP_GAME/USRDIR/DATA.PAK`, containing all original function names, variable names, source filenames, and line numbers. These were generated by the SN Systems ProDG toolchain, confirming the game was compiled with **SNC**.

Why this target:

- **Small scope.** PSP Minis are hard-capped at 100 MB, with no multiplayer and no DLC. A racing game's logic (track loop, physics tick, render) tends to be straightforward.
- **Full debug symbols.** Every function and variable name is known. The system can be validated against ground truth end-to-end.
- **US region.** Easy to source, English-language strings for agent comprehension.
- **SNC compiler.** A working SNC binary (`pspsnc 1.2.7503.0`) is available from the decompme/compilers repository. This is a Windows PE executable runnable via wibo on Linux/macOS.
- **ViciousEngine.** Shares engine lineage with multiple full-size commercial games, enabling direct pipeline transfer to Milestone 2.
- **Completely untouched.** No existing decompilation work anywhere — no GitHub repos, no decomp.me scratches, no forum threads.

Top Gun (USA, PSP Mini) is a viable backup. It is also a ViciousEngine Mini with full debug symbols and zero prior work, but flight/combat logic is likely more complex than racing.

### Milestone 2: Full ViciousEngine game (undecided)

The leading candidate is **Alien Syndrome** (USA, PSN), a full commercial action RPG published by Sega and developed by Totally Games (2007). It is ~274 MB, has 40 levels, co-op multiplayer, and RPG stat/gear systems. It also ships with `.sym` and `.map` debug symbols in DATA.PAK and uses the same ViciousEngine / SNC toolchain as Days of Thunder.

This is an unusual opportunity: a real commercial game that still has debug symbols. The system graduates from "small with answers" to "big with answers" while reusing the entire pipeline — same engine, same compiler, same symbol format, same archive structure.

Other ViciousEngine full games with debug symbols (Ben 10: Alien Force, Despicable Me, DON 2) are fallback options. The final choice will be made after Milestone 1 validates the pipeline.

### Milestone 3: ViciousEngine game without symbols (future)

Several ViciousEngine games ship without debug symbols: Dead Head Fred (Vicious Cycle's flagship, ~500 MB action-adventure), 300: March to Glory, Puzzle Quest. These use the same engine and compiler but provide no ground truth. This is the true test — proving the system can produce a matching decompilation with no human involvement and no answers to check against.

The specific target will be chosen after Milestone 2 based on what the system has learned about ViciousEngine's code patterns and the SNC compiler's behavior.

## Why ViciousEngine

No other PSP engine offers this combination:

1. **Shared engine across games of varying complexity** — from 100 MB Minis to 500 MB action-adventures.
2. **Debug symbols in retail builds** — both Minis and full games, providing ground truth at every milestone.
3. **Single compiler (SNC)** — no per-game compiler identification needed within the engine family.
4. **Completely untouched** — zero existing decompilation work on any ViciousEngine title.
5. **Graduation path with increasing difficulty** — Mini with symbols, full game with symbols, full game without symbols.

## Compiler details

ViciousEngine games were compiled with SNC (SN Systems Compiler), the proprietary compiler from the ProDG SDK. This is confirmed by the `.sym` files being in SN Systems ProDG format.

Available compiler binary: `pspsnc 1.2.7503.0` from [decompme/compilers](https://github.com/decompme/compilers). This is the only publicly available SNC version. An early pipeline task must verify this version produces matching output against Days of Thunder — if it doesn't, the correct version will need to be identified.

Three compilers existed for PSP commercial development:
- **SNC** (SN Systems / ProDG) — dominant, used by ~83% of launch titles
- **GCC** (Sony official SDK) — baseline toolchain, versions 3.3.x and 4.0.x
- **MWCCPSP** (Metrowerks CodeWarrior) — used by Capcom, Koei, and others

Compiler identification for non-ViciousEngine targets requires forensic analysis: `cplinit` ELF section indicates SNC, `.comment` section strings indicate GCC or MWCC, single-section MWLD layout indicates CodeWarrior.

## Other targets considered

| Target | Verdict | Reason |
|--------|---------|--------|
| ViciousEngine Minis (Top Gun, Cart Kings, etc.) | Backup for M1 | Same qualities as Days of Thunder |
| BOOT.BIN symbol games (Puzzle Bobble, Rengoku, Armored Core) | Rejected | Japanese region, unknown compiler, no engine lineage to POC, no graduation path |
| EA EAGL games (NFS ProStreet, Medal of Honor) | Rejected | EA internal engine, likely different compiler, complex codebases, no Mini equivalent |
| FFT: War of the Lions | Rejected | No debug symbols, uncertain compiler, complex, no graduation path from POC |
| Castlevania: SotN PSP | Rejected | Already being actively decompiled (sotn-decomp) |
| Non-ViciousEngine full games with symbols | Deferred | No shared engine with any Mini; would require separate pipeline validation |

## Risks

- **SNC version mismatch.** The available `pspsnc 1.2.7503.0` may not be the version used to compile Days of Thunder. Mitigation: test early by compiling a known function and comparing bytes.
- **SNC compiler flags unknown.** Optimization level, flag combinations, and include paths must be determined empirically. The .sym files may contain hints (source filenames, compilation unit boundaries).
- **DATA.PAK extraction.** The archive format needs to be reverse-engineered or an existing extractor found. ViciousEngine's PAK format may not be publicly documented.
- **PRX modules.** While most ViciousEngine games likely ship as monolithic ELFs, this should be verified for each target. If Days of Thunder loads PRX modules at runtime, those must also be decompiled.
- **EBOOT decryption.** EBOOT.BIN is encrypted. Tools like pspdecrypt handle this, but the decryption step must be integrated into the pipeline.
